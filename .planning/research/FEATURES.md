# Project Research: Features

## Table Stakes

- A `SKILL.md` that clearly states when to use the skill and what role the agent is playing.
- A disciplined inquiry loop from disturbance to examined proposition.
- Dialogue rules that keep the agent asking one substantive question at a time.
- Candidate-belief handling that makes uncertainty explicit and requires user selection or revision.
- A question taxonomy broad enough to support real Socratic examination.
- Safety boundaries that interrupt philosophical inquiry for crisis or diagnosis requests.
- A session summary template that records the artifact of inquiry.
- A session schema for tracking state across turns.

## Differentiators

- Pathologos pattern references that help agents notice recurring hidden-belief shapes without asserting certainty.
- A belief graph template for future recurring-proposition analysis.
- A validation script that checks structured session artifacts.
- An evaluation rubric for scoring session quality.

## Anti-Features

- Generic reassurance as a default response.
- Interpretations presented as certain knowledge of the user's unconscious.
- Philosophical essays that replace inquiry.
- Advice-giving, coaching, therapy, diagnosis, or crisis counseling.
- Premature CBT-style reframing.

## Dependencies

- `SKILL.md` depends on the reference docs being precise and scoped.
- The validator depends on a stable session schema.
- The summary template depends on the schema and dialogue loop.
