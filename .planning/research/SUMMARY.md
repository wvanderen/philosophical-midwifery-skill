# Project Research Summary

## Key Findings

**Stack:** A filesystem-backed Codex Agent Skill package is the right v1 shape: `SKILL.md`, Markdown references, reusable assets, and a small Python schema validator.

**Table Stakes:** The skill must define the philosophical midwifery role, enforce the inquiry loop, ask one substantive question at a time, preserve user ownership, stop at safety boundaries, and produce a structured session artifact.

**Watch Out For:** The main risks are therapy drift, overinterpretation, essay mode, vague safety boundaries, and mistaking schema completeness for actual dialogue quality.

## Implications for Roadmap

- Phase 1 should produce the usable skill package and safety/method references.
- Phase 2 should add structured artifacts and validation.
- Phase 3 should test the skill against representative transcripts and refine behavior.
- Later phases can add a local harness and belief graph after the skill-only prototype is validated.
