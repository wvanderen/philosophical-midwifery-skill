# Question Taxonomy

This is your reference for selecting the next examination question. Read it when the user has confirmed a proposition and you need to decide which Socratic operation to apply next. You ask one question at a time; this file helps you choose *which* one.

**Voice note (soft modals).** The guidance here uses soft modals — *prefer, try, consider*. The work of examination is judgment, not rule-following, so keep your flexibility in live dialogue. This contrasts with `SAFETY_BOUNDARIES.md`, which uses hard modals (MUST / NEVER) because the cost of non-compliance is harm.

**Ground rules, restated.** Ask ONE substantive question at a time. Prefer questions over assertions. Keep the inquiry anchored to the user-confirmed proposition. The operations below are lenses, not a script — pick the one the proposition's current state calls for.

## The eight examination operations

### Clarification

- **What it does:** Defines a vague or loaded term before it can be tested.
- **When to use it:** Use this when the proposition turns on a term whose meaning isn't yet fixed — words like *worth*, *failure*, *recognition*, *love* — so that later moves don't slide past an unexamined meaning.
- **Soft guidance:** Try asking the user to give the term in their own words, or to give a concrete instance of it.
- **Example questions:**
  - "What do you mean by *worth*?"
  - "When you say *recognition*, what specifically counts — a promotion? A thank-you? Being seen?"
  - "What exactly would count as *failure* here?"

### Universalization

- **What it does:** Tests whether the claim applies generally, not just to the user.
- **When to use it:** Use this once the proposition is stated and its key terms are defined, and you want to expose whether the user is applying a rule to themselves that they would not apply to others.
- **Soft guidance:** Prefer phrasing that places another person in the same situation.
- **Example questions:**
  - "Would this be true for everyone, or only for you?"
  - "If a colleague's excellent work went unnoticed, would that make it worthless?"
  - "Does this rule apply to other people, or are you an exception?"

- **Worked snippet — "recognition determines worth":**
  > **Agent:** "If a colleague did excellent work and no one noticed it, would that make their work worthless?"
  > **User:** "No… their work would still be good."
  > **Agent:** "Then what makes your own case different?"

### Implication

- **What it does:** Exposes what must follow if the belief is true.
- **When to use it:** Use this when the proposition is clear and you want to trace its consequences — sometimes the user endorses a belief but rejects what it implies.
- **Soft guidance:** Try "If that's true, then what else follows?"
- **Example questions:**
  - "If recognition determines worth, what does that imply about work that is never seen?"
  - "What would this belief mean for people who never receive recognition?"
  - "If we accept this, what else has to be true?"

### Counterexample

- **What it does:** Finds a concrete case that weakens the proposition.
- **When to use it:** Use this when the proposition seems too sweeping and a single lived instance might show it does not always hold.
- **Soft guidance:** Consider inviting the user to supply the case themselves — their own examples carry more weight than yours.
- **Example questions:**
  - "Can you think of something valuable that went completely unrecognized?"
  - "Have you ever respected someone whose work failed?"
  - "Is there a case where someone was rejected but not diminished?"

### Necessity / sufficiency

- **What it does:** Tests logical dependency — whether X is *required*, or *enough*, for Y.
- **When to use it:** Use this when the proposition claims a dependency ("worth depends on recognition") and you want to probe whether the dependency is necessary, sufficient, both, or neither.
- **Soft guidance:** Try splitting the claim in two — "Is it necessary? Is it sufficient?" — and take them one at a time.
- **Example questions:**
  - "Is recognition *necessary* for worth — can worth exist without it?"
  - "Is recognition *sufficient* — does being recognized guarantee worth?"
  - "Can someone be recognized and still lack merit?"

### Domain separation

- **What it does:** Separates concepts the user may have fused.
- **When to use it:** Use this when the proposition conflates two distinct domains — visibility and value, status and worth, judgment and truth — such that a claim about one is being treated as a claim about the other.
- **Soft guidance:** Prefer naming the two domains explicitly and asking whether they are the same.
- **Example questions:**
  - "Are we talking about *visibility* here, or *value*? Are they the same?"
  - "Is *social status* the same as *worth*?"
  - "Is *being noticed* the same as *matteringing*?"

### Contradiction test

- **What it does:** Identifies two commitments the user holds that cannot both be true.
- **When to use it:** Use this when the user has affirmed something earlier that conflicts with the proposition now under examination — a live inconsistency to surface.
- **Soft guidance:** Present both commitments plainly and ask whether they can coexist; let the user choose which to keep.
- **Example questions:**
  - "You said valuable things can go unrecognized — but you also said that being overlooked means your work has no value. Can both be true?"
  - "Does this belief conflict with what you said earlier?"
  - "Which of these two claims do you want to hold onto?"

### Reformulation

- **What it does:** Invites a more coherent version of the proposition after pressure.
- **When to use it:** Use this once a contradiction or weakness has surfaced and the user is ready to restate the belief more carefully.
- **Soft guidance:** Consider asking the user to do the reformulation themselves — a revised proposition they authored will hold better than one you handed them.
- **Example questions:**
  - "Given what we found, what version of this belief survives?"
  - "How would you state this more carefully now?"
  - "What is still true once we set aside the part that didn't hold?"

## Worked example — "recognition determines worth" (full arc)

The shared example threading this reference with `../SKILL.md` and the other references:

> **Proposition (user-confirmed):** "The value of my work depends on recognition."
>
> **Clarification:** "What do you mean by *recognition*? And what do you mean by *value*?"
> **Universalization:** "If a colleague's excellent work went unnoticed, would it be worthless?"
> **Necessity / sufficiency:** "Is recognition *necessary* for value? Is it *sufficient*?"
> **Domain separation:** "Are *visibility* and *value* the same thing?"
> **Contradiction test:** "You've said unnoticed work can still be good — yet you feel yours has no value when unnoticed. Can both hold?"
> **Reformulation:** "Given all this, how would you restate what recognition actually does to your work?"

The aim is a revised proposition the user endorses — e.g., "Recognition affects visibility and opportunity, but does not determine the worth of the work itself" — alongside any tension that remains unresolved.
